Лабораторная работа 1: Исследование компилятора GCC, язык ассемблера, Makefile, Git
Тема
Исследование компилятора GCC, трансляция программы на языке C в ассемблерный код с различными уровнями оптимизации, разработка модульной структуры и Makefile, реализация параллельного процесса с синхронизацией.

1. Исходная программа
Реализована программа вычисления чисел Фибоначчи на языке C. Функция fib_iter вычисляет число Фибоначчи итеративным методом, fib_rec — рекурсивным. Программа принимает два аргумента командной строки: n (номер числа) и метод вычисления (i или r).

2. Трансляция в ассемблер
Получены ассемблерные листинги с различными уровнями оптимизации:

bash
gcc -S -O0 -o fibonacci_O0.s fibonacci.c
gcc -S -O1 -o fibonacci_O1.s fibonacci.c
gcc -S -O2 -o fibonacci_O2.s fibonacci.c
gcc -S -O3 -o fibonacci_O3.s fibonacci.c
gcc -S -Os -o fibonacci_Os.s fibonacci.c
Проведён анализ ассемблерного кода для уровней оптимизации -O0, -O2 и -O3:

Уровень	Особенности
-O0	Все переменные хранятся в стеке, код максимально читаемый
-O2	Переменные хранятся в регистрах (%eax, %ecx, %edx), цикл оптимизирован
-O3	Агрессивное развёртывание рекурсии, функция fib_iter заинлайнена в main
Сравнение размеров:

Уровень	Строк кода
-O0	89
-O2	67
-O3	183
Версия -O3 оказалась самой большой из-за развёртывания рекурсивной функции fib_rec в итеративный блочный алгоритм.

3. Анализ ассемблерного кода (-O2)
Подробно разобрана функция fib_iter с оптимизацией -O2:

Переменные: n — в %edi, a — в %ecx, b — в %rax, i — в %edx

Цикл for: реализован через метку .L4 и условный переход jne .L4

Условие n <= 1: проверка через cmpl $1, %edi и jle .L7

Обмен значений: выполняется через временный регистр %rsi (tmp = b)

Оптимизация условия цикла: вместо i <= n используется i != n+1 (быстрее)

В функцию fib_iter добавлен swap — обмен значений переменных a и b с использованием ассемблерной инструкции xchg.

4. Модульная структура и Makefile
Программа разбита на модули:

text
fib_project/
├── fib.h            — заголовочный файл
├── fib_iter.c       — итеративное вычисление Фибоначчи
├── fib_rec.c        — рекурсивное вычисление Фибоначчи
├── main.c           — главный модуль
└── Makefile         — файл сборки
Makefile содержит цели:

all — сборка программы

asm — генерация ассемблерных листингов с разными флагами оптимизации

clean — очистка скомпилированных файлов

5. Параллельный процесс
Создана параллельная версия программы с использованием fork() и синхронизацией через pipe:

Родительский процесс вычисляет fib(40)

Дочерний процесс вычисляет fib(45)

Результат передаётся от дочернего процесса родителю через канал pipe

Родительский процесс ожидает завершения дочернего через wait()
